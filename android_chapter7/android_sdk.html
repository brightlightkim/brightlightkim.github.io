ActivityOptions

This is the part to create the animation.
Example:
cheatButton.setOnClickListener(new View.OnClickListener() {
@Override
public void onClick(View view) {
// Start CheatActivity
boolean answerIsTrue =
getQuizViewModel().getCurrentQuestionAnswer();
Intent intent = CheatActivity.newIntent(MainActivity.this,
answerIsTrue);
ActivityOptions options =
ActivityOptions.makeClipRevealAnimation(
view, 0, 0, view.getWidth(), view.getHeight());
startActivityForResult(intent, REQUEST_CODE_CHEAT,
options.toBundle());
}
});

Better Approach

Add @SuppressLint("RestrictedApi")

if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
ActivityOptions options =
ActivityOptions.makeClipRevealAnimation(
view, 0, 0, view.getWidth(), view.getHeight());
startActivityForResult(intent, REQUEST_CODE_CHEAT,
options.toBundle());
} else {
startActivityForResult(intent, REQUEST_CODE_CHEAT);
}

Jetpack Libraries

Guarding API is not the optimal
1. It's more work for me as a developer.
2. It means that users of your app will have a different experience based on the version their device is running

Jetpack Libaries offer new features (like ViewModel), backward compatibility for new features on older devices and provide consistent behavior when possible across Android versions.
It allows you to write a few conditional API checks as possible.
