Main Activity

AppCompatActivity

subclass of Android's Activity class that provides compatibility support for older versions of Android.

onCreate

it called when an instance of the activity subclass is created

You can link the activity to its UI by using
Activity.setContentView(layoutResID: Int).
This function inflates a layout and puts it onscreen.
When a layout is inflated, each widget in the layout file is instantiated as defined by its attributes.
You specify which layout to inflate by passing in the layout's resource ID.


Resources and Resource IDs

Layout is a resource. A resource is a piece of your application that is not code thing like image files, audio files, and XML files.
Resources for your project live in a subdirectory of the app/res directory.
To access a resource in code, you use its resource ID.
The resource ID for your layout is R.layout.activity_main


R

R is the place where stores all of the resource ID for the entire layout and for each string.
However, it does not generate resource IDs for the individual widges in activity_main.xml

To generate resource ID

you include an android:id= "@+id/true_button"
There is a +sign for android:id but not in the values for android:text.
This is because you are creating the resource IDs and only referencing the strings
